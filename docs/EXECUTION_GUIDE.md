# Pipeline Execution Guide

To keep the different stages of the project conceptually and computationally isolated, I separated the entire codebase into three primary orchestrator scripts. You have to run them in a very specific, sequential order because the later machine learning stages strictly depend on the massive `.parquet` datasets generated by the physics engine.

### Stage 1: Theoretical Data Synthesis
The first thing you need to run is the physics generator.
```bash
python physics_main.py
```
This is the heaviest computational step. What happens here is that the solver spins up multiple CPU workers to evaluate the Tolman-Oppenheimer-Volkoff equations over a vast grid of theoretical equations of state. It utilizes both the Anchored Speed-of-Sound model for hadronic matter and the Generalized CFL model for quark matter. The script essentially builds a massive catalog of hypothetical stars and saves all their macroscopic observables (mass, radius, tidal deformability) into `data/physics_test_dataset.parquet`. If you need to generate a massive dataset for final evaluation, it might take a few hours depending on how high you set `ATTEMPT_MULTIPLIER` in the config.

### Stage 2: Ideal Machine Learning Classification
Once the `.parquet` file is written to disk, you can train the models.
```bash
python main.py
```
This script loads up the theoretical dataset and trains the primary classifiers, specifically XGBoost and the PyTorch Bayesian Neural Networks (BNN). Because this data is "clean"—meaning it represents the exact mathematical output of the TOV equations without any observational error—the classifiers should easily learn to distinguish the quark phase from the hadronic phase. The script will output the trained model weights into the `models/` directory, and it automatically drops all the diagnostic plots (like SHAP feature importance and Brier calibration curves) into `plots/`. 

### Stage 3: Observational Noise and the Masquerade Problem
This is the most critical part of the thesis methodology.
```bash
python perturb_main.py
```
In real astrophysics, we never measure a neutron star's mass and radius perfectly; telescopes like NICER have significant error bars. Because hybrid stars and pure hadronic stars can have nearly identical mass-radius curves—a degeneracy heavily discussed in literature known as the "Masquerade Problem" (e.g., Alford et al.)—we have to prove our model works under real-world conditions. 

This script takes the clean dataset and aggressively smears the mass and radius columns with Gaussian noise to simulate those telescopic uncertainties. It then retrains the classifiers. The results plotted in `plots_perturb/` definitively prove that when mass and radius are obscured by noise, the inclusion of the tidal deformability parameter $\Lambda$ (which is extracted from gravitational wave interferometers like LIGO) is absolutely mandatory to break the degeneracy and maintain high classification accuracy.
